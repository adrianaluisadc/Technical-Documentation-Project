What is CSS?

So what really is it?
For a little history trivia, it was developed by World Wide Web Consortium (W3C) and had its initial release on December 17, 1996 (almost 24 years ago). Today, CSS3 is the latest version of the CSS specification that supports responsive layout.
But before we embark our journey in learning the basics of CSS, one thing I have to share first is that CSS is NOT a programming language.
Cascading Style Sheets or CSS is actually a style sheet language and not a programming language because unlike the latter, CSS only let you set styles for your HTML elements in the web pages, including design, layout and display variations for various devices and screen sizes but it DO NOT gives instruction and commands. In other words, it does not program!

Types of CSS

There are three ways to integrate CSS in the HTML file and they are:

  • Inline
  • Internal
  • External

Inline Styles

To make it simple, Inline CSS is when your are integrating a CSS style in the line of your HTML document on a specific element. If you have set a style in your Internal or External CSS (these two will be discussed later), the inline CSS will override them.

Example:

<h1 style="color: fuchsia"; font-weight: bolder; text-decoration: underline> This is an inline style </h1>

If we preview the HTML file with this inline css style on the Chrome browser, this style will look like this:

Internal Styles

Internal CSS styles are styles inside the HTML file and it is easy to identify them because they are enclosed within a <style> tag. Internal style will only apply to the HTML document it is written. It is suggested to put the style tag inside the head tag for HTML validation rather than inside the body tag.

Example:

<head> 
  <style> 
    h1 {
    color: green;
    font-family: Arial;
    }
  </style>
</head> 
<body> 
  <h1>This is an Internal Style!</h1>
</body>

Here is the preview of the style in the Chrome browser:

External Styles

This type of style is recommended and highly suggested. This style can be added to your HTML file by utilizing the <link> tag to link a CSS file. The rel attribute of the link tag must be set to "stylesheet" and the href attribute to the stylesheet's relative or absolute path. In this scenario, the type attribute of the link element is not required. For the HTML to load the elements style first, you must place the link tag inside the head tag.

Example:

index.html

<!DOCTYPE html> 
<html>
  <head>
    <meta charset="utf-8">
    <link rel="stylesheet" type="text/css" href="style.css"> 
  </head> 
  <body> 
    <h1>This is an External Style...</h1>
  </body>
</html>

style.css

h1 {
  color: blue;
  font-style: italic;
  font-weight: bolder;
  font-family: Impact;
}

The style will look like this in the browser:

Note: I have created a css file inside the same folder where the HTML file is. Keep in mind that you need to properly direct the link tag to the right path for it to find the CSS file.

Anatomy of CSS ruleset

We'll look at the parts that make up the CSS as we know it on this topic. With no further ado, the rules of the CSS has three parts: the selector, the property, and the value.

Selector

The selector is a pointer to the HTML element you're styling.

Examples of selectors: h1, p, #idname, .classname, body, *, section, main

Property

CSS properties styles the selector where it is nested. There are a lot of CSS properties and you can check this link to see a basic list of them.

Examples of properties: color, text-decoration, display, font-weight, background, line-heigh, margin

Value

The value specifies a specific styling of the property.

Examples of properties with values: margin: auto, line-height: 1em, color: #000, font-family: Arial

Comments in CSS

Commenting in CSS is also important especially if you are working on a big project. Comments would help you remember what the styles are for and it helps you organize your css file.

Single Line Comment

/* A sample of single line comment*/
div {
  font-size: 1em; /* This is a single line comment */
}
